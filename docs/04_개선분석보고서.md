**목적:** NeoPMS / WATCH2-Project / WATCH2Defect 벤치마크 분석 결과에서 도출한 **기능 갭 보완** 항목을 식별하고, 요구사항정의서·기능구조도에 반영할 개선안과 개발환경 적용사항을 정리한다.

## 1. 개선 방향 요약

벤치마크 비교표는 두 제품의 강점을 "채택" 관점에서 정리했으나, 다음 세 축의 빈틈이 남아 있다. 본 보고서는 이를 **갭 보완 중심**으로 메운다.

1. **공통 갭(Both-Weak)** — 두 제품 모두 △ 또는 ✕ 인 영역. uplusitPMS가 채워야 할 1순위이자 차별화 기회.
2. **반영 누락(Under-spec)** — 한쪽 제품의 강점이지만 현 SRS v0.1에서 우선순위가 낮거나 명세가 얕은 항목.
3. **차별화(Beyond-benchmark)** — 두 제품을 단순 통합하는 것을 넘어 uplusitPMS만의 경쟁력을 만드는 신규 기능.

개선의 핵심 초점은 **기능 갭 보완**에 두며, AI·자동화와 공공/망분리 특화는 갭을 메우는 수단으로 함께 다룬다.

## 4. 차별화 기능 (Beyond-benchmark)

두 제품의 단순 합집합을 넘어 uplusitPMS의 경쟁력을 만드는 신규 영역. 모두 **폐쇄망 내에서 동작**하는 것을 전제로 한다.

**D1. 계획↔실행 단일 데이터 브리지** — 비교표·SRS의 선언적 목표("계획 Waterfall + 실행 Agile을 하나의 데이터로")를 실제 기능으로 구현한다. WBS Task ↔ 칸반 작업카드를 양방향 동기화하여, 어느 쪽에서 진척을 입력해도 S-Curve·번다운에 동시 반영되도록 한다. 두 벤치마크 제품은 WBS(WATCH2 강점)와 칸반(NeoPMS 강점)을 각각 보유하지만 **연결하지 못한다** — 이 연결이 핵심 차별점이다. (`REQ-BRG-01~02`)

**D2. 폐쇄망 AI 어시스턴트(RAG)** — 외부 SaaS 의존 없이 온프레미스 sLLM + RAG로, 프로젝트 산출물·표준·이력을 근거로 한 자연어 질의응답, 회의록 요약, 요구사항–테스트 누락 탐지를 제공한다. 공공 망분리 환경에서 작동하는 AI는 두 제품 모두 부재한 영역이다. (`REQ-AI-01~05`)

**D3. 공공 감리 대응 자동화** — 감리 지적사항 관리대장, 표준 산출물 누락 자동 점검, 행안부 표준 채번·도메인 준수 검증을 제공한다. NeoPMS의 메타관리와 WATCH2의 방법론 거버넌스를 결합해 **감리 준비를 자동화**한다. (`REQ-AUD-01~02`)

**D4. 통합 RTM(End-to-End)** — 요구 → 분석(유스케이스) → 설계(화면/API/테이블) → 구현(프로그램) → 테스트(케이스) → 결함의 전 구간을 단일 매트릭스로 추적한다. NeoPMS·WATCH2Defect는 부분 RTM만 제공하므로, 전 구간 커버리지 자동 산출이 차별점이 된다. (REQ-TST-07 확장)

## 5. 개발환경 적용사항 (Technical Stack 권고)

벤치마크 제품의 기술 구성과 공공/망분리 제약을 종합한 권고. 설계 단계 확정 전 검토 기준으로 사용한다.

| 영역 | NeoPMS | WATCH2 | uplusitPMS 권고 | 근거 |
|------|--------|--------|------------------|------|
| 백엔드 | eGovFrame 계열 | (미공개) | **Spring Boot + eGovFramework** | 공공 표준 프레임워크 적합성 |
| DB | PostgreSQL | (PostgreSQL 추정) | **PostgreSQL** | 양 제품 공통·국산화 라이선스 부담 적음 |
| 프론트엔드 | WebSquare(상용) | 웹/모바일 | **표준 SPA(React 또는 Vue) + 반응형** | 상용 RIA 라이선스 회피, 모바일 갭(G3) 해소 |
| 호환성 | Non-ActiveX·HTML5 | 웹표준 | **Non-ActiveX·HTML5 표준 준수** | 공공정보화 지침 |
| 배포 | Tomcat 단독 | **망분리 Docker 단일배포** | **Docker Compose 단일배포 + 화이트라벨** | WATCH2 강점 채택, 폐쇄망 반입 용이 |
| 메신저 | 업무쪽지(자체) | Rocket.Chat SSO | **Rocket.Chat 온프레미스 연동 + 업무쪽지 병행** | 폐쇄망 설치형, SSO 연동 검증된 구성 |
| AI/LLM | AI Reporting(단편) | 없음 | **온프레미스 sLLM + RAG(폐쇄망)** | 외부 API 차단 환경 대응, 차별화(D2) |
| 인증 | 토큰 기반 | 토큰·세션 | **토큰 기반 + 세션 타임아웃 + 사이트 라이선스 서명** | NFR-03 충족 |

**미결 검토사항**
- 소스 제너레이터(REQ-DEV-04): 내재화 vs 외부도구 연계 — 폐쇄망 반입성 고려해 **경량 내재화** 우선 검토.
- 온프레미스 LLM 모델·GPU 사양 산정: 발주처별 인프라 제약에 따라 모델 등급(경량/표준) 분기.
- WebSquare 등 상용 RIA 채택 시 라이선스 비용 vs 표준 SPA 개발공수 트레이드오프 정량 비교 필요.

## 7. 결론

NeoPMS와 WATCH2의 강점 통합만으로는 두 제품의 **공통 약점(AI·통합알림·모바일·EVM·연동관리)**이 그대로 남는다. uplusitPMS는 이 공통 갭을 메우는 동시에, 계획↔실행 단일 데이터 브리지(D1)와 폐쇄망 AI 어시스턴트(D2), 공공 감리 자동화(D3)를 차별화 축으로 삼아 "통합을 넘어선 진보"를 달성해야 한다. 다음 단계로 요구사항정의서 v0.2 개정과 기능구조도 보강에 본 개선안을 반영한다.
