# A. 시스템 아키텍처

## A.1 설계 원칙
- **단일 진실 공급원(SSOT):** 진척·품질·이슈 데이터는 한 곳에 저장하고 화면은 이를 집계만 한다.
- **모듈러 모놀리스:** 망분리·단일 배포(NFR-05)에 유리하도록 단일 디플로이 + 명확한 바운디드 컨텍스트. 추후 모듈 분리(MSA) 가능하게 경계 유지.
- **이벤트 기반 크로스커팅:** 진척 브리지·알림·감사·AI는 도메인 이벤트로 느슨하게 결합.
- **폐쇄망 우선:** 외부 SaaS 의존 0, 모든 의존성 오프라인 번들로 반입(P3).
- **표준 준수:** 공공정보화 표준 도메인·행안부 채번(NFR-08), Non-ActiveX·HTML5(NFR-04).

## A.2 기술 스택 (확정안)
| 레이어 | 채택 | 사유 |
|--------|------|------|
| 프론트엔드 | React 18 + TypeScript, Vite, TanStack Query, 차트(ECharts/Chart.js) | 표준 SPA·라이선스 free, 위젯/간트/칸반 생태계 |

## A.4 크로스커팅 설계
- **진척 브리지 엔진(UC-BRG-01):** WBSTask·WorkCard 변경 시 `ProgressChanged` 도메인 이벤트 발행 → 브리지가 단일 진척으로 정규화 → S-Curve/번다운/EVM/대시보드 캐시 갱신. 동시수정은 낙관적 락(version) + 감사로그.
- **통합 알림(UC-NTF-01):** 모듈 이벤트 → 알림 디스패처가 구독 매핑 조회 → 인앱(WebSocket/polling)·메일·RC 채널 라우팅 → 알림 이력 저장.
- **감사(REQ-SEC-02):** JPA `@EntityListeners`/AOP로 모든 CUD를 `audit_logs`에 적재(행위자·대상·전후값).
- **인증/인가:** JWT 필터 → RBAC 권한 평가(역할·모니터링 읽기전용) → 메서드 보안(`@PreAuthorize`).
- **멀티프로젝트:** 모든 업무 엔티티에 `project_id` 스코프, 행 수준 접근통제.
- **국제화/표준:** 공통코드·메타표준 기반 도메인 검증, 행안부 채번 시퀀스.

## A.5 배포 아키텍처 (망분리)
```
[운영 서버(폐쇄망)]
 docker-compose:
   ├ uplusitpms-web      (React 정적 + Nginx)
   ├ uplusitpms-api      (Spring Boot)
   ├ postgres            (볼륨 영속)
   ├ redis
   ├ rocketchat (+mongo) (선택)
   └ slm-server          (선택, GPU 노드)
 install.sh / backup.sh / restore.sh · .env · 라이선스 키(.lic)
 오프라인 번들: images.tar(docker save) + compose + 시드데이터
```
설치 절차는 P4 설치 가이드에서 상세화(사전요건→이미지 load→compose up→마이그레이션→검증).

# B. 데이터 설계 (ERD)

## B.1 명명 규칙
- 테이블: 소문자 복수형 스네이크(`wbs_tasks`). PK: `id`(BIGINT, IDENTITY). FK: `<참조단수>_id`.
- 공통 컬럼: `created_at, created_by, updated_at, updated_by`(감사), `version`(낙관적 락), 논리삭제 `deleted_at`(필요 모듈).
- 멀티프로젝트 스코프 컬럼: `project_id`(업무 엔티티 공통).
- 코드성 값: `codes`(공통코드) 또는 ENUM 문자열 + 검증.

# C. API 명세

## C.1 공통 규약
- **Base URL:** `/api/v1` · **포맷:** JSON(UTF-8) · **인증:** `Authorization: Bearer <accessToken>`.
- **프로젝트 스코프:** 헤더 `X-Project-Id` 또는 경로 `/projects/{projectId}/...`.
- **메서드:** GET(조회)·POST(생성)·PUT/PATCH(수정)·DELETE(삭제). 목록은 `page,size,sort,q,filter`.
- **응답 포맷:**
```json
{ "success": true, "data": { ... }, "page": {"page":0,"size":20,"total":135}, "error": null }
```
- **오류:** HTTP 4xx/5xx + `{ "success":false, "error":{ "code":"REQ_BASELINE_LOCKED", "message":"..." } }`.
- **공통 코드:** 400 검증오류, 401 미인증, 403 권한없음, 409 충돌(낙관적 락/상태전이 위반), 422 업무규칙 위반.

## D.2 핵심 화면 명세

### S-01 대시보드 (UC-DSH-01)
- **레이아웃:** 캐치프레이즈 배너 + 위젯 그리드(드래그 재배치).
- **위젯:** WBS달성률·품질달성도·단계별진척률·이슈/결함처리율·오늘의작업·D-Day·최근공지.
- **액션:** 위젯 클릭→상세 드릴다운, 위젯 토글/이동(개인화 저장).
- **상태:** 로딩(스켈레톤)/빈데이터/권한제한. **데이터:** `GET /dashboard`.

### S-02 WBS·일정 (UC-SCH-01,02 / UC-BRG)
```
[탭: 트리 | 간트 | 캘린더]   [+행추가][엑셀▼][일괄지연처리]

└─────────────────────┘└───────────────────────────────┘
```

### S-04 요구사항·베이스라인 (UC-RQM-01,02)
- **레이아웃:** 통계카드(전체/상태분포/유형) + 요구 목록 + 상세 패널.
- **액션:** 등록·엑셀 일괄, "수용"건 일괄 **베이스라인 확정**, 변경 시 사유·근거·승인자 모달(필수), RTM 보기.
- **상태:** 베이스라인 잠금 후 변경은 변경관리 경로만 허용. **데이터:** `/requirements`, `/baseline`, `/changes`.

### S-05 테스트·결함 (UC-TST-02,03)
```
[시나리오 패널]│[케이스 목록]              [수행결과▼]

 IT-002 …      │ UT-001-02  실패 →[결함등록(멀티)]
               │   심각도[중] 유형[기능] 📎클립보드캡처
```
- **액션:** 수행결과(성공/실패/보류/제외), 멀티결함 등록, 결함조치/비결함 검토, 재시험.
- **상태:** 성공/실패 케이스 수정잠금, 자기검증 차단. **데이터:** `/executions`, `/defects`, `/retest`.

### S-06 산출물 결재 (UC-QLT-02)
- **레이아웃:** 산출물 목록 + 결재선 타임라인(PL→QA→PMO→발주처).
- **액션:** 상신, 단계별 승인/반려(사유), 본인 대기건 일괄 승인, 확정 시 폴더 잠금.
- **상태:** 반려→재제출, 확정→잠금 배지. **데이터:** `/approvals`, `PATCH /approval-lines/{id}`.

### S-07 주간/월간 보고 (UC-COM-02)
- **레이아웃:** 보고 워크플로 진행바(생성→작성요청→작성중→검토→완료) + 팀별 섹션 + 자동수집 데이터.
- **액션:** 생성·작성요청(알림), 팀 섹션 작성, 취합·검토, 완료, DOCX 다운로드.
- **데이터:** `/reports`, `PATCH /reports/{id}/workflow`, `GET /reports/{id}/docx`.

### S-08 통합 알림함 (UC-NTF-01)
- **레이아웃:** Top Bar 🔔 → 드롭다운(최근) + 전체 알림함(필터: 결재/지연/결함/멘션).
- **액션:** 읽음 처리, 알림 클릭→원본 화면 이동, 수신 채널 구독 설정.
- **데이터:** `GET/PATCH /me/notifications`, `/notification-subscriptions`.
